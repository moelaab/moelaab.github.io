Welcome to Academic!

Berikut langkah-langkah menghubungkan komputer Windows ke GitHub menggunakan SSH, disesuaikan dengan sistem operasi Windows 11 (seperti yang Anda gunakan):

1. Pastikan OpenSSH Client sudah terinstal

  • Buka Settings → Apps → Optional features
  • Cari OpenSSH Client
    • Jika belum ada, klik Add a feature → cari OpenSSH Client → install

Alternatif: Gunakan Git Bash (jika sudah menginstal Git for Windows), karena sudah menyertakan SSH.

2. Buka terminal (PowerShell, Command Prompt, atau Git Bash)

Disarankan gunakan Git Bash (lebih mirip Linux) atau PowerShell.

3. Buat pasangan kunci SSH

Jalankan perintah berikut di terminal (ganti your_email@example.com dengan email GitHub Anda):

ssh-keygen -t ed25519 -C "your_email@example.com"

Jika muncul error seperti “unknown key type”, gunakan RSA:

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
  • Tekan Enter untuk menyimpan di lokasi default (C:\Users\<NamaUser>\.ssh\id_ed25519)
  • Masukkan passphrase (opsional tapi aman) atau tekan Enter lagi untuk melewati

4. Mulai SSH Agent (di Windows)

Jalankan di PowerShell sebagai admin (jika diperlukan):

# Mulai layanan SSH Agent
Get-Service -Name ssh-agent | Set-Service -StartupType Automatic
Start-Service ssh-agent

Lalu tambahkan kunci privat:

ssh-add ~/.ssh/id_ed25519

Jika pakai Git Bash, jalankan langsung di Git Bash.
Jika pakai PowerShell dan kunci bernama id_rsa, ganti sesuai nama file.

5. Salin kunci publik ke clipboard

Jalankan di terminal:

cat ~/.ssh/id_ed25519.pub

Atau, jika pakai PowerShell:

Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard

Ini akan menyalin seluruh isi kunci publik ke clipboard.

6. Tambahkan kunci publik ke GitHub

  1. Buka https://github.com/settings/keys
  2. Klik New SSH key
  3. Isi:
    • Title: misalnya Windows 11 Laptop
    • Key type: pilih Authentication Key
    • Key: tempelkan kunci yang sudah disalin
  4. Klik Add SSH key

7. Uji koneksi SSH ke GitHub

Jalankan di terminal:

ssh -T git@github.com

Jika muncul peringatan seperti:

Are you sure you want to continue connecting (yes/no)?

Ketik yes lalu tekan Enter.

Jika berhasil, Anda akan melihat:

Hi <username>! You've successfully authenticated...

8. Gunakan URL SSH saat clone atau remote

Saat meng-clone repositori, pastikan gunakan format SSH:

git clone git@github.com:username/nama-repo.git

Untuk mengubah remote yang sebelumnya HTTPS ke SSH:

git remote set-url origin git@github.com:username/nama-repo.git

✅ Setelah ini, Anda bisa git push/pull tanpa memasukkan username & password GitHub.

Jika Anda menggunakan VS Code atau GitHub Desktop, pastikan mereka juga menggunakan terminal/Git yang mendukung SSH (biasanya otomatis terdeteksi).

Semoga membantu!

Install Python and JupyterLab

Install Anaconda which includes Python 3 and JupyterLab.

Alternatively, install JupyterLab with pip3 install jupyterlab.

Create or upload a Jupyter notebook

Run the following commands in your Terminal, substituting <MY-WEBSITE-FOLDER> and <SHORT-POST-TITLE> with the file path to your Academic website folder and a short title for your blog post (use hyphens instead of spaces), respectively:

mkdir -p <MY-WEBSITE-FOLDER>/content/post/<SHORT-POST-TITLE>/
cd <MY-WEBSITE-FOLDER>/content/post/<SHORT-POST-TITLE>/
jupyter lab index.ipynb

The jupyter command above will launch the JupyterLab editor, allowing us to add Academic metadata and write the content.

Edit your post metadata

The first cell of your Jupter notebook will contain your post metadata (front matter).

In Jupter, choose Markdown as the type of the first cell and wrap your Academic metadata in three dashes, indicating that it is YAML front matter:

---
title: My post's title
date: 2019-09-01

# Put any other Academic metadata here...
---

Edit the metadata of your post, using the documentation as a guide to the available options.

To set a featured image, place an image named featured into your post’s folder.

For other tips, such as using math, see the guide on writing content with Academic.

Convert notebook to Markdown

jupyter nbconvert index.ipynb --to markdown --NbConvertApp.output_files_dir=.
